# Инструкция по начальной настройке

## Назначение доработок

В конфигурацию добавлены механизмы:
- обслуживания клиентов по абонентским договорам;
- массового создания актов;
- управленческого расчета зарплаты;
- планирования отпусков;
- отчетов по новым данным.

## Что должно быть в базе до начала работы

Перед использованием новой функциональности в базе должны быть заведены:
- организация;
- контрагенты;
- договоры контрагентов;
- пользователи системы;
- номенклатура для абонентской платы;
- номенклатура для работ специалиста.

Перед началом работы также рекомендуется проверить:
- заполнены ли карточки пользователей, которые будут работать как специалист, менеджер, бухгалтер и кадровик-расчетчик;
- существует ли пользователь `Администратор` с полными правами.

## Настройка договоров на абонентское обслуживание

Для договоров, которые будут использоваться в обслуживании клиентов, необходимо:
1. Открыть справочник договоров контрагентов.
2. Создать новый договор или открыть существующий.
3. Установить вид договора `ВКМ_АбонентскоеОбслуживание`.
4. Заполнить:
	- дату начала действия;
	- дату окончания действия;
	- сумму ежемесячной абонентской платы;
	- стоимость часа работы специалиста.
5. Записать договор.

Без этих данных документы обслуживания и автоматическое создание актов не будут работать корректно.

## Заполнение констант

Перед использованием автоматического заполнения реализаций необходимо заполнить константы:
1. `ВКМ_НоменклатураАбонентскаяПлата`
2. `ВКМ_НоменклатураРаботыСпециалиста`
3. `ВКМ_ПроцентНДФЛ`

Порядок заполнения:
1. Открыть форму констант или соответствующие формы настройки.
2. Указать номенклатуру для строки абонентской платы.
3. Указать номенклатуру для строки работ специалиста.
4. Указать процент НДФЛ, используемый при расчете зарплаты.

## Настройка прав пользователей

Пользователям необходимо назначить профили групп доступа в зависимости от выполняемых функций.

Минимально должны быть настроены:
- `Менеджер по продажам` для пользователя, который планирует обслуживания;
- `Обслуживание клиентов пользователь` для пользователя-специалиста;
- `Главный бухгалтер` или профиль, включающий права `ВКМ_БухгалтерИТФирмы`, для бухгалтера;
- `КадровикРасчетчик` для пользователя, который ведет условия оплаты, начисления, выплаты и отпуска;
- `Администратор` для технического пользователя сопровождения.

После назначения профилей пользователя рекомендуется переподключить к базе.

## Настройка зарплатного блока

Для расчета зарплаты необходимо заполнить регистр `ВКМ_УсловияОплатыСотрудников`.

Для каждого сотрудника указываются:
- период действия записи;
- оклад;
- процент от работ.

Для административного персонала процент от работ может не использоваться.
Для специалистов процент от работ должен быть заполнен, иначе документы обслуживания не будут проводиться.

## Базовый порядок работы с обслуживанием клиентов

1. Менеджер создает документ `ВКМ_ОбслуживаниеКлиента`.
2. В документе указываются:
	- клиент;
	- договор;
	- специалист;
	- дата проведения работ;
	- описание проблемы;
	- комментарий.
3. Специалист открывает документ и заполняет табличную часть `ВыполненныеРаботы`.
4. Провести документ.
5. После проведения документа данные попадают в регистры и используются в актах, зарплате и отчетах.

## Массовое создание актов

Для автоматического создания актов бухгалтер должен:
1. Открыть обработку `ВКМ_МассовоеСозданиеАктов`.
2. Указать период.
3. Нажать `Заполнить`.
4. Проверить список договоров.
5. Нажать `Создать реализации`.

Если реализация за выбранный месяц уже существует, повторно она не создается.

## Расчет зарплаты

Для расчета зарплаты используются:
- `ВКМ_НачисленияЗарплаты`;
- `ВКМ_НачислениеФиксированнойПремии`;
- `ВКМ_ВыплатаЗарплаты`;
- `ВКМ_ВводОтпуска`;
- `ВКМ_ГрафикОтпусков`.

Общий порядок работы:
1. Заполнить условия оплаты сотрудников.
2. Ввести документы обслуживания клиентов.
3. При необходимости ввести отпуск и фиксированные премии.
4. Создать документ начисления зарплаты.
5. Создать документ выплаты зарплаты.

## Отчеты

После ввода и проведения документов пользователям доступны отчеты:
- `ВКМ_ВыработкаСпециалистов`;
- `ВКМ_АнализВыставленныхАктов`;
- `ВКМ_РасчетыССотрудниками`;
- `ВКМ_РасходЗапланированныхОтпусков`.

Перед формированием отчетов необходимо убедиться, что все используемые документы проведены.

## Автоматизированный тест

В репозитории подготовлен сценарий Vanessa Automation:
- `tests/features/01_Менеджер_ПланированиеОбслуживаний.feature`

Для запуска сценария необходимо:
1. Открыть `tests/vanessa-automation-single.epf`.
2. Указать файл сценария:
	- `tests/features/01_Менеджер_ПланированиеОбслуживаний.feature`
3. Убедиться, что в базе существуют пользователи:
	- `Менеджер`
	- `Специалист 2`
	- `Бухгалтер`
4. Убедиться, что в базе существуют данные:
	- `Клиент 1`
	- `Договор 1`
	- `Специалист 1`

## Контрольная проверка после настройки

После выполнения начальной настройки рекомендуется вручную проверить:
1. Создание документа `ВКМ_ОбслуживаниеКлиента`.
2. Проведение документа и появление движений.
3. Массовое создание актов.
4. Формирование отчета `ВКМ_АнализВыставленныхАктов`.
5. Создание документа начисления зарплаты.
